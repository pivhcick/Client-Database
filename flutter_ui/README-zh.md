
## 目录详细说明

#### `lib/`

- **components/**  
  存放可复用的组件，组件数据来源于设计稿中的组件，组件集。

- **custom_widgets/**  
  存放设计图页面中较为复杂的节点代码，把较复杂节点提取出来作为单独widget，避免大量代码全部堆积在page里面，增加代码可读性。

- **utils/**  
  存放pixso生成的一些工具类，屏幕适配类等。

- **variables/**  
  存储所有变量信息，可以获取图层应用变量值、更新图层变量集模式。


#### `assets/`
存放静态资源（如图片、字体）。

项目的说明文档，介绍目录结构，代码结构。

## 代码说明
1. 代码规范遵循组件+实例，生成的时候识别到图层是实例，会引用相关组件，达到代码简洁以及提升代码可用性。

2. 在生成的代码中，会遇到属性是变量。例如 `VariablesManager().getString("xxx")`。变量规则可以参考设计稿规则。

3. 在设计稿转换为代码过程中，遇到复杂的矢量图层或者复杂的图层表现形式中，例如图片采用了裁剪模式，我们会将该图层转化为图片。

4. 我们支持复杂的原型交互，例如页面跳转，点击事件，以及滚动方向支持。其他原型交互进一步支持中。

5. 如果遇到生成的代码编译错误，请联系产品客服，我们会提供远程解决方案。
